# 2026-03-11 回滚方案（数据与兼容）

## 目标

当新版本出现严重问题（崩溃/无法读书/无法入库/缓存任务异常）时，能够安全回滚到上一稳定版本，并明确哪些数据可能需要清理。

## 关键数据位置（概念）

- 网文数据库：`webnovel.db`（schema version 当前为 `3`）
- 网文章节缓存表：`web_chapter_cache`（含 `last_accessed_at`、`size_bytes`）
- 网文缓存下载任务：`web_download_tasks`、`web_download_settings`
- 运行日志：`run.log`（含 `RUN_EVENT` 结构化事件行）

具体落盘目录取决于平台（Windows/Android），优先以应用设置页或运行日志导出路径为准。

## 兼容性说明（向后兼容）

- `webnovel.db` 的升级为“增量/加列/加表”模式，旧版本通常可忽略新列（有默认值）并继续读取旧数据。
- 若回滚后出现“目录/缓存任务/会话”异常，建议清理“缓存任务记录/缓存内容”，尽量避免直接删除书架数据。

## 回滚步骤（建议顺序）

1. 导出运行日志（用于定位问题）：设置页 → 运行日志 → 导出/分享
2. 卸载或覆盖安装到上一版本安装包（Android APK / Windows installer）
3. 若回滚后仍异常：
   - 先在“网文缓存管理”清理：暂停全部 → 清理已完成/失败 → 清空任务
   - 仍异常再清空缓存章节（会影响离线阅读）
4. 最后手段：删除 `webnovel.db`（会丢失网文会话/历史/缓存索引，书架书籍本体通常不受影响）

## 需要特别注意

- 回滚前尽量保留 `webnovel.db`，除非明确确认数据库损坏或新旧版本冲突导致启动失败。
- 若问题与“书源规则/站点兼容”相关，优先导出书源包与导入报告，避免仅靠回滚掩盖问题。
