# 文文Tome 角色化解决方案报告（2026-03-11）

> 目标：稳定性与阅读体验对齐搜书大师 + 夸克式网页阅读闭环，完成 AI 半自动能力与回溯体系，并形成发布闭环。  
> 原则：严格区分书源搜索/网页搜索；AI 仅“建议/影子验证”；所有变更可回滚可审计。

---

## 1. 问题清单与落地摘要（对齐 10 条）

1. **大型 TXT 卡死**：引入超大 TXT 懒加载（首屏预览 + 后台全量加载）、目录延迟生成、运行日志埋点。
2. **EPUB 冷启动转圈**：持久化解析缓存 + 失败回退 + 生命周期取消保护，冷启动优先走缓存。
3. **点击区域误触**：点击区域逻辑统一（全屏菜单/中间菜单分区），音量键翻页去重并可关闭。
4. **翻页卡顿**：分页预热 + 耗时埋点 + 新增仿真二（TurnPageView）与现有仿真优化。
5. **字体乱码**：字体名与 UI 文案清洗，导入/展示层过滤“�”。
6. **书源搜索混入非小说**：默认关闭网页兜底；仅在用户开启时触发并标记“网页兜底”。
7. **搜书结果过少/不持续增长/未聚合**：流式搜书 + 聚合展示 + 详情补齐 + 持续增长计数。
8. **网页阅读闭环缺失**：识别增强 → 阅读模式 → 加书架 → 缓存 → 目录闭环落地。
9. **AI 接入评估 + 回溯机制**：AI 搜书重排/过滤、AI 书源修复（影子验证）+ 版本表回滚。
10. **角色化解决方案报告**：本报告与任务清单同步交付，保证交接可续。

---

## 2. 角色化分工与执行视角

### 2.1 10 年经验全栈工程师（主程）
**职责重点**
- 架构级收敛：书源搜索/网页搜索强隔离、reader_mode 虚拟书源、缓存/版本/日志闭环。
- 性能与稳定性：大文件懒加载、EPUB 缓存、分页预热、AppStabilityMonitor。
- AI 工程化：接入已有 API 配置、可回滚、可审计、可灰度。

**关键动作**
- 统一 `RUN_EVENT` 关键路径埋点，补齐慢/卡/闪退观察。
- 读写层隔离：文本加载、缓存 DB、书源 DB 独立演进。
- reader_mode 采用虚拟书源策略，不改变 BookFormat，减少架构风险。

### 2.2 10 年经验产品经理（主 PM）
**职责重点**
- 闭环流程定义：网页搜索 → 识别 → 阅读模式 → 加书架 → 缓存 → 目录。
- 验收标准明确：每条问题给出可量化验收（时间/次数/成功率）。
- 风险与发布节奏：Android-only patch，Windows 仅同步版本号。

**关键动作**
- 拆分“严格书源搜索 vs 网页搜索兜底”用户心智，文案明确。
- 设定 AI 规则：默认关闭、影子验证后可应用、回滚入口直达。

### 2.3 新入职工程师（工程师 A）
**职责重点**
- UI/交互落地：搜书流式列表、聚合详情、阅读模式按钮组。
- 交互细节：点击区域/音量键行为一致性。

**关键动作**
- WebNovel 页新增“进入阅读模式/加入书架/缓存/目录/原网页”入口。
- 阅读页顶部补充缓存/原网页入口。

### 2.4 新入职工程师（工程师 B）
**职责重点**
- 数据与缓存：reader_mode 章节生成、web_chapter_cache 写入、版本表回滚。
- 工具链：release_check 版本一致性、Android 打包脚本。

**关键动作**
- reader_mode 章节列表生成（TOC heuristics + 当前页兜底）。
- AI 书源修复：解析 AI patch → 影子验证 → 版本化应用。

### 2.5 新入职产品经理（PM）
**职责重点**
- QA 回归用例编写、录屏复现标准化。
- 与用户沟通“AI 仅建议/影子验证”机制，建立信任。

**关键动作**
- 录制前后对比复现视频，整理“搜书大师/夸克”对齐点。
- 维护任务清单状态：已实现/待验证/风险点。

## 3. 分阶段落地步骤（M1–M5）

### M1 稳定性与阅读性能（P0）
**交付**
- TXT 懒加载 + 后台补齐 + 目录延迟
- EPUB 解析缓存 + 生命周期取消保护
- 预分页 + 翻页动画优化（仿真一/仿真二）
- 点击区域/音量键统一逻辑
- 字体/文案清洗

**验证**
- 50/100MB TXT 打开 <5s
- EPUB 冷启动 5 次无转圈
- 翻页连续不卡顿（真机）

### M2 搜书链路对齐（P0）
**交付**
- 搜书流式结果 + UI 节流刷新
- 结果聚合（同书多源合并）
- 严格书源搜索 + 网页兜底开关
- 详情补齐（封面/简介/作者/状态/字数）

**验证**
- 搜书结果持续增长，不等全部完成
- 非小说不混入

### M3 网页阅读闭环（P0/P1）
**交付**
- detectReaderMode 增强（作者/目录/下一章）
- reader_mode 虚拟书源（无书源 fallback）
- 浏览器动作闭环（进入阅读模式/加书架/缓存/目录/原网页）

**验证**
- 识别成功率、目录可用
- 缓存任务可在管理页追踪

### M4 AI 能力与回溯（P1）
**交付**
- AI 搜书重排 + 非小说过滤
- AI 书源修复（建议/影子验证）
- `web_source_versions` 版本表 + 一键回滚

**验证**
- 影子验证失败不应用
- 版本回滚可恢复旧规则

### M5 质量与发布闭环（P1）
**交付**
- `RUN_EVENT` 关键路径补齐
- 版本号/更新日志/发布说明同步
- Android-only 出包

**验证**
- `scripts/release_check.ps1 -SkipWindows` 通过
- APK 输出到 `releases/2.6.11`

---

## 4. 风险与回滚策略

- **AI 误修复**：只在影子验证通过后应用；所有变更写入版本表，可一键回滚。
- **网页识别误判**：仅“阅读模式”入口主动触发；保留原网页入口。
- **性能回退**：分页优化与缓存均可通过开关与日志快速定位。
