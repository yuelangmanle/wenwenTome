# 给下一位开发者的提示词（可直接复制）

你现在接手 `wenwen_tome` 项目，请按以下顺序推进，不要先做无关优化。

## 1) 先阅读这些文档（必须）
1. `docs/qa/2026-03-09-project-progress-book.md`
2. `docs/qa/2026-03-09-soushu-full-reference-and-copy-report.md`
3. `docs/plans/2026-03-09-soushu-reference-implementation-tasklist.md`
4. `CHANGELOG.md`（看 `2.2.0` 与 `2.3.0`）

## 2) 当前最高优先级（按顺序）
1. 修“加入书架后未获取网络章节”：做二阶段拉目录任务、失败重试、状态回写。
2. 修网页搜索触控：WebView 可正常滑动/点击/缩放，不再只能按钮翻页。
3. 修阅读性能与章节：超长 TXT 不再卡死，EPUB 章节名与跳转稳定。
4. 修 TTS 稳定性：下载模型后朗读不闪退，并在阅读页可切换引擎。
5. 完成阅读点击策略：支持“中间菜单+左右翻页”和“全屏菜单”两模式。

## 3) 实施约束
- 可以借鉴搜书大师的架构、流程、数据模型与配置思想。
- 不要复制反编译代码，不要引入法律风险实现。
- 先保可用性，再做重构；每次只收敛一条主链路。
- 所有关键失败路径必须有可读错误文案和恢复动作。

## 4) 交付要求
- 每完成一个子模块，必须给出：
  1. 改动文件清单
  2. 可复现测试步骤（真机）
  3. 风险与回归点
  4. 下一个最优先任务

## 5) 验证命令
- `flutter analyze`
- `flutter test`
- Android 真机回归最少覆盖：
  - 网文：搜书 -> 加书架 -> 打开章节 -> 缓存后离线读
  - 阅读：TXT/EPUB 跳章、拖进度、切分页模式
  - TTS：切换引擎、连续朗读、异常恢复

## 6) 你这轮结束时必须更新
- `CHANGELOG.md`（写清用户可见变化）
- `docs/qa/2026-03-09-project-progress-book.md`（更新最新进度）
- 如新增方案，补到 `docs/plans/2026-03-09-soushu-reference-implementation-tasklist.md`
