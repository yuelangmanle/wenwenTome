## 2. 关键现状说明（重要）

### 2.1 为什么“书源搜索”会出现非小说内容
- 当前在 `searchBooksWithReport` 中，当直连书源不足时会触发“搜索引擎兜底”。
- 兜底逻辑来自 `_searchSourceByProviderFallback` / `_searchSourcesByProviderFallbackBulk`。
- 这相当于把“网页搜索”混进了“书源搜索”，导致知乎/百科等非小说结果混入。

### 2.2 必须做的策略分离
- 书源搜索必须提供 “严格仅书源” 模式（默认开启）。
- 网页搜索独立入口，兜底仅在用户明确允许时触发。

## 4. 详细任务清单（极其详细）

> 每条任务包含：范围、交付物、验收、风险/回滚点。

### 4.1 稳定性与性能（M1）

**M1-01 大 TXT 流式加载与首屏优先**
- 交付物：流式解码+首屏渲染；目录/索引异步补齐
- 验收：50/100MB TXT 打开 ≤ 5s，首屏稳定可读
- 风险：解码准确率（UTF‑16/GBK/UTF‑8 兼容）

**M1-02 EPUB 冷启动恢复**
- 交付物：解析缓存+失败回退+生命周期安全
- 验收：冷启动 5 次均可进入阅读（无转圈）
- 风险：内容为空、章节/正文映射异常

**M1-03 翻页卡顿专项**
- 交付物：分页缓存 + 预分页 + 渲染耗时埋点
- 验收：连续翻页不卡顿，FPS≥55（真机）
- 风险：内存涨、分页缓存过大

**M1-04 交互统一**
- 交付物：点击区域模式 A/B；菜单/翻页冲突消除
- 验收：全屏菜单任意点唤起；中间菜单左/中/右分区翻页
- 风险：与音量键翻页冲突

**M1-05 字体乱码排查**
- 交付物：字体名称清洗 + 全量 UI 文字扫描
- 验收：设置页/阅读页/提示无乱码
- 风险：系统字体名称编码不一致

### 4.2 搜书体验对齐（M2）

**M2-01 搜书结果流式增长**
- 交付物：结果分批实时刷新，顶部计数持续增长
- 验收：结果呈“源源不断”增长，不等待全部完成
- 风险：UI 刷新频率过高

**M2-02 搜书结果聚合**
- 交付物：同书聚合（title/author/别名），来源数量展示
- 验收：同一本书聚合展示，详情页可查看多来源
- 风险：误聚合、漏聚合

**M2-03 结果去噪与严格书源搜索**
- 交付物：书源搜索默认不使用网页兜底
- 验收：书源搜索结果仅来自书源解析
- 风险：结果数量下降 → 提示“允许兜底”开关

**M2-04 详情页持续补齐**
- 交付物：封面/简介/作者/分类/状态持续补齐与标注来源
- 验收：详情页信息稳定完整，来源切换不丢
- 风险：异源字段冲突

**M2-05 章节列表与下载提示**
- 交付物：章节列表分页/“显示全部章节”/下载状态提示
- 验收：详情页出现“下载内容中…第X章”
- 风险：章节索引错乱

### 4.3 网页阅读闭环（M3）

**M3-01 网页内容识别**
- 交付物：识别目录/详情/章节页
- 验收：站点页能判断是否可读
- 风险：站点差异大

**M3-02 阅读模式抽取**
- 交付物：正文抽取、章节标题、分页、下一章链接
- 验收：阅读视图稳定进入，正文无广告
- 风险：抽取规则不稳定

**M3-03 加书架/缓存/原网页**
- 交付物：阅读页固定入口 + 缓存剩余章节对话框
- 验收：与夸克体验一致
- 风险：目录识别失败导致缓存异常

### 4.4 AI 能力与回溯（M4）

**M4-01 AI 搜书增强**
- 交付物：语义排序 + 非小说过滤 + 命中提升
- 验收：同关键词搜书命中率显著提高
- 风险：模型成本/延迟

**M4-02 AI 书源修复**
- 交付物：失败规则自动生成修复建议 + 验证脚本
- 验收：修复命中率与回归通过率可量化
- 风险：误修复

**M4-03 AI 接管策略**
- 交付物：建议模式 / 半自动 / 自动模式
- 验收：自动改动必须可回滚
- 风险：灰度失败

**M4-04 回溯机制**
- 交付物：书源版本化 + 影子验证 + 一键回滚
- 验收：任意变更可追踪、可恢复
- 风险：存储与管理成本

### 4.5 质量与发布闭环（M5）

**M5-01 回归集与测试流程**
- 交付物：TXT/EPUB/弱网/翻页/缓存/阅读模式回归集
- 验收：一键回归通过
- 风险：样本集维护成本

**M5-02 发布检查单**
- 交付物：版本号 + Changelog + 回滚说明 + release 脚本
- 验收：发布流程可重复
- 风险：版本回滚兼容问题
