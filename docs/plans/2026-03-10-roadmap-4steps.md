# 2026-03-10 四步推进清单（建议执行顺序）

目标：把 `wenwen_tome` 收敛成「可稳定使用」的版本：网文可搜可入库可缓存、阅读器稳、TTS 稳、发布闭环可重复。

说明：
- 每一步都包含：范围（对应任务号）、交付物、验收（DoD）、回归清单、风险与回滚要点。
- 建议严格按顺序做：先把最影响可用性的“缓存下载”做成可控服务，再补齐格式缺口与阅读性能，随后完善质量与发布闭环，最后处理翻译模块策略收敛。

## Step 1：缓存下载服务化（WDL-001 ~ WDL-008）

### 1.1 范围
- WDL-001 拆分书级任务与章节级任务队列
- WDL-002 任务落库（queued/running/paused/failed/completed）
- WDL-003 失败原因结构化（HTTP/解析/限流/规则）
- WDL-004 断点续传能力统一（以“章节粒度可恢复”为主）
- WDL-005 后台恢复（App 重启后任务继续）
- WDL-006 缓存管理页（批量暂停/继续/清除）
- WDL-007 缓存覆盖策略（重复章节更新规则）
- WDL-008 空间配额与清理策略（按书/按时间）

### 1.2 交付物（必须全部具备）
- 任务系统（书级任务 → 展开为章节级任务；章节级负责实际下载）
- 任务持久化（DB 表：任务、设置；启动恢复 running → queued）
- 失败原因与重试策略（指数退避；原因分类用于 UI/排障）
- 并发与开关（最大并发可配置；可暂停/继续）
- 缓存管理页（任务列表、并发配置、空间策略、清理按钮）
- 缓存清理策略（按 quota/maxBooks/maxDays 触发清理；清理后任务一致性维护）

### 1.3 规则/策略明确化（写死在代码与文档里）
- 去重：同一本书/同一章节重复入队时必须“可预测”（忽略重复或合并到同一个任务）
- 覆盖：章节已缓存时，允许“覆盖更新”或“跳过”，但必须一致、可解释
- 恢复：暂停/重启后继续从“剩余章节”开始，不要求 HTTP Range 级别续传
- 清理：清理必须按书/时间可控，且不会破坏 DB 一致性（孤儿任务/孤儿缓存）

### 1.4 验收（DoD）
- 功能：
  - 从网文书籍页触发“缓存章节/预热”，UI 不阻塞，任务进入队列并可在管理页看到
  - 可“暂停全部/继续全部”；暂停后不会继续产生新下载
  - App 重启后，未完成任务能继续（至少能从 queued 继续跑完）
  - 可清除任务（终态/全部），可清除缓存（全量/按书）
- 质量：
  - `flutter test` 全绿
  - `flutter analyze` 无新增 error
  - DB 升级不丢数据（旧缓存仍可读；新增字段默认值正确）
- 可观测：
  - 失败任务在列表中可见，并能展示“失败原因分类 + 最近错误信息”

### 1.5 回归清单（建议自动化 + 手工）
- 自动化：任务/缓存相关单测 + 迁移测试（见 Step 3 的 REL/QAT）
- 手工：
  1. 同一本书入队两次：观察是否去重/合并，且不会爆炸式增长
  2. 弱网/断网：任务进入 failed，并能 resume 后继续
  3. 设置 quota/maxBooks/maxDays：缓存增长后能触发清理，且阅读不报错

### 1.6 风险与回滚要点
- DB 版本升级不可逆：必须确保 `onUpgrade` 的默认值与新表创建可重复执行
- 回滚时要保证旧版本仍能读取缓存表（新增列需带默认值）
- 清理策略必须保守：宁可少删也不要误删“当前正在读”的缓存（用 last_accessed_at）

## Step 2：格式链路缺口 + 阅读目录跳转性能（FRM-007/008、RDR-004）

### 2.1 范围
- FRM-007 PDF 目录和页码映射统一
- FRM-008 MOBI/AZW3 自动转换链路稳定化（失败可解释）
- RDR-004 目录跳转性能优化（避免全量重排）

### 2.2 交付物
- PDF：目录项 → 页码映射一致；跳转与页码显示一致；异常文件有清晰报错
- MOBI/AZW3：转换失败给出明确原因与可操作建议；转换结果可缓存复用
- RDR：目录跳转不触发全量重排/长时间卡顿；跳转后位置稳定

### 2.3 验收（DoD）
- 提供最少 3 类样本：
  - 复杂 PDF（多级目录/混合页面尺寸）
  - 典型 MOBI/AZW3（含图片/含目录/无目录）
  - 长文本阅读（大章节、目录多）
- 新增/更新回归用例（能稳定复现、可自动跑）

### 3.1 范围
- QAT-001~007：样本集、弱网回归、埋点与日志标准化、超时/取消策略统一

### 3.2 交付物

- Release 脚本可复用（环境变量/缓存目录固定；失败可诊断）
- 回滚说明（DB 向后兼容点与不兼容点列表）

### 3.3 验收（DoD）

  - `flutter analyze`、`flutter test`
  - 最小 smoke（启动、打开书、跳章、TTS、网文缓存）
- 发布材料齐备：版本号、Changelog、用户可见更新说明、回滚说明
